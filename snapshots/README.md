# snapshots

这个目录用于记录全局规则的重要历史版本或阶段性备份，便于回退、比对和审计。

## 建议放什么

- `AGENTS-YYYYMMDD-HHMMSS.md`
- `global-AGENTS-YYYYMMDD-HHMMSS.md`
- 其他与全局规则直接相关的快照文件

## 建议何时新增快照

- 大幅调整协作规则时
- 变更技能触发策略时
- 修改安全确认机制时
- 准备删除多个项目级 `AGENTS.md` 之前
- 准备同步到 GitHub 前后，需要留档时

## 命名建议

- 文件名尽量包含对象名和时间戳
- 时间格式统一使用 `YYYYMMDD-HHMMSS`
- 如需说明用途，可加简短前缀，例如 `before-cleanup-`、`before-publish-`

## 维护规则

- 快照目录以“可回退、可核对”为目标，不追求复杂结构。
- 快照应尽量保留原文，不要在快照文件里混入大量解释性内容。
- 需要解释变更原因时，优先在提交信息或单独说明文件中写清楚。
